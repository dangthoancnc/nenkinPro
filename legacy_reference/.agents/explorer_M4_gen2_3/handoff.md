# Handoff Report: Form Generator Vulnerabilities

## 1. Observation
1. **Broken Access Control**: Investigated `src/app/api/generate-form/route.ts`. The `POST` function does not verify if the user is authenticated. There is no usage of `cookies()` or checks for the `employee_auth` cookie which is standard across other endpoints like `src/app/api/customers/route.ts`.
2. **Path Traversal**: In `src/app/api/generate-form/route.ts:25`, the template path is generated by concatenating `process.cwd()`, `'public'`, `'templates'`, and `templateName` directly: `const templatePath = path.join(process.cwd(), 'public', 'templates', templateName);`. This allows a malicious user to supply a `templateName` like `../../some-file` to perform directory traversal.
3. **HTTP Header Encoding Crash**: In `src/app/api/generate-form/route.ts:59`, the `Content-Disposition` header directly interpolates `templateName`: `'Content-Disposition': \`attachment; filename="Generated_${templateName}"\``. When `templateName` contains Japanese or non-ASCII characters, it fails to parse as a valid header value and throws a ByteString conversion error.
4. **DOM Cleanup**: In `src/app/customers/page.tsx:151-152`, an anchor element is created and appended to the DOM to trigger a download (`document.body.appendChild(a); a.click();`). However, there is no corresponding `document.body.removeChild(a);` to clean it up afterwards, leading to hidden anchor tags cluttering the DOM with each download.

## 2. Logic Chain
- **Broken Access Control Fix**: To restrict access only to logged-in employees, `src/app/api/generate-form/route.ts` needs to import `cookies` from `next/headers` and verify that `cookieStore.get('employee_auth')?.value` exists, similar to how it is handled in `src/app/api/customers/route.ts`. If not, return a 401 Unauthorized response.
- **Path Traversal Fix**: Using `path.basename(templateName)` will extract only the filename component from the provided `templateName`, discarding any directory traversal sequences (e.g., `../`). This sanitized value should be used in `path.join()`.
- **HTTP Header Encoding Fix**: Standard HTTP headers strictly require ASCII characters. To support Japanese filenames properly according to RFC 5987, the `Content-Disposition` header should use the `filename*` parameter encoded in UTF-8: ``filename*=UTF-8''Generated_${encodeURIComponent(sanitizedTemplateName)}``.
- **DOM Cleanup Fix**: Adding `document.body.removeChild(a);` immediately after `a.click();` in `src/app/customers/page.tsx` will properly clean up the DOM element while the download process remains unaffected.

## 3. Caveats
- The fix for `Content-Disposition` assumes the generated file will just be the sanitized template name prefixed with "Generated_".
- Ensure that `path` is correctly imported in the API route (it already is on line 3 of `src/app/api/generate-form/route.ts`).
- Did not modify or implement the code directly as per constraints, only providing the required fix strategies and paths.

## 4. Conclusion
Four distinct issues were identified across two files and the fix strategy is as follows:

**File 1: `src/app/api/generate-form/route.ts`**
- **Import update**: Add `import { cookies } from 'next/headers';`.
- **Access Control**: At the top of the `POST` handler, add:
  ```typescript
  const cookieStore = cookies();
  const userId = cookieStore.get('employee_auth')?.value;
  if (!userId) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });
  }
  ```
- **Path Traversal**: Change line 25 to:
  ```typescript
  const sanitizedTemplateName = path.basename(templateName);
  const templatePath = path.join(process.cwd(), 'public', 'templates', sanitizedTemplateName);
  ```
- **HTTP Header**: Change line 59 to:
  ```typescript
  'Content-Disposition': `attachment; filename*=UTF-8''Generated_${encodeURIComponent(sanitizedTemplateName)}`,
  ```

**File 2: `src/app/customers/page.tsx`**
- **DOM Cleanup**: Add `document.body.removeChild(a);` after `a.click();` around line 152 in the `handleGenerateForm` function.

## 5. Verification Method
1. Start the application locally and verify that a request to `/api/generate-form` without the `employee_auth` cookie returns a 401 error.
2. Send a request to `/api/generate-form` with `templateName` set to `../../some-file.docx` and ensure the API successfully sanitizes it and looks for `some-file.docx` in the correct directory.
3. Attempt to generate a form using a template name containing Japanese characters (e.g. `テスト.docx`). Verify that the file downloads without crashing the server and retains its Japanese characters.
4. Trigger a download on the `src/app/customers/page.tsx` page and inspect the DOM via browser developer tools to ensure no stale `<a>` tags remain appended to `<body>`.
